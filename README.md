# Resource Hub 资源导航站

Resource Hub 是一个轻量级 PHP 资源导航站，适合用于公众号自动回复、AI 工具合集、电脑工具合集、软件下载页、博客资源页和私域引流承接。

项目无需数据库，使用 JSON 文件存储数据，部署简单，下载后即可运行。内置后台管理功能，支持资源新增、编辑、删除、分类管理、图标上传、首页展示和站点信息配置。

## 项目特点

- 无需数据库，使用 JSON 文件存储
- 部署简单，上传到 PHP 环境即可运行
- 自带后台登录页面
- 支持资源新增、编辑、删除
- 支持资源分类管理
- 支持关键词搜索
- 支持工具图标上传
- 支持多个访问按钮，例如官网、下载、教程、备用链接
- 支持站点名称、描述等基础配置
- 适合公众号、博客、工具站、AI 资源库使用
- 适合做私域引流承接页

## 使用场景

### 1. 公众号自动回复资源页

适合公众号运营者把工具、教程、下载地址统一整理到一个页面。

例如：

```text
用户关注公众号
↓
回复关键词【工具包】
↓
自动回复 Resource Hub 资源页
↓
用户进入页面获取工具、教程、下载链接
```

适合用于：

- 电脑工具合集
- AI 工具合集
- 开源项目整理
- 软件安装教程
- 公众号资料包领取页

---

### 2. 博客资源中心

如果你有自己的博客，可以把 Resource Hub 作为博客的资源中心页面。

例如：

↓
文章底部引导：更多工具见资源中心
↓
跳转 Resource Hub
↓
用户浏览更多资源

适合用于：

- 技术博客资源库
- 下载中心
- 教程合集页
- 常用网站导航
- 开源项目推荐页

## 快速部署

### 1. 下载项目

```bash
git clone https://github.com/QiuQiuZhen/resource-hub.git
```

或者直接下载 ZIP 压缩包，上传到服务器网站目录。

---

### 2. 上传到服务器

例如使用宝塔面板：

```text
网站目录：/www/wwwroot/resource-hub
运行环境：PHP 7.4 或 PHP 8.x
数据库：不需要
```

---

### 3. 设置目录权限

确保以下目录具有写入权限：

```text
data/
uploads/
uploads/icons/
```

Linux 服务器可执行：

```bash
chmod -R 755 data uploads
```

如果后台无法保存数据或上传图标，可以尝试：

```bash
chmod -R 775 data uploads
```

---

### 4. 访问前台

```text
https://你的域名.com/
```

或者：

```text
https://你的域名.com/index.php
```

---

### 5. 访问后台

```text
https://你的域名.com/admin.php
```

默认测试账号：

```text
账号：admin
密码：change_me_123456
```

> 默认账号仅用于本地测试和快速体验，正式部署后请立即修改后台账号和密码。

## 资源数据说明

项目使用 JSON 文件保存资源数据。

资源数据一般存放在：

```text
data/tools.json
```

站点设置一般存放在：

```text
data/settings.json
```

这样做的好处是：

- 不需要安装数据库
- 迁移方便
- 备份简单
- 小型资源站足够使用

## 正式部署建议

正式上线前，建议完成以下操作：

### 1. 修改默认后台账号密码

打开：

```text
config.php
```

修改默认账号和密码。

不要继续使用：

```text
admin
admin123456
```

改成更安全的账号和强密码。

---

### 2. 关闭目录浏览

如果使用 Nginx，请确保没有开启目录浏览。

如果使用 Apache，可以在敏感目录中添加 `.htaccess` 限制访问。

例如保护 `data` 目录：

```apache
Deny from all
```

---

### 3. 备份数据文件

建议定期备份：

```text
data/tools.json
data/settings.json
uploads/icons/
```

这些文件保存了你的资源内容和上传图标。

---

### 4. 不要上传敏感信息

如果你二次开发了接口，请不要把以下内容提交到公开仓库：

```text
真实后台密码
真实 API Key
真实 Token
```

## GitHub Pages 说明

本项目包含 PHP 后台接口，因此不能仅依靠 GitHub Pages 完整运行。

GitHub Pages 只适合托管静态页面，不支持 PHP 后台功能。

如果只上传到 GitHub Pages，可能只能看到部分前端页面，无法使用：

- 后台登录
- 数据保存
- 图标上传
- 接口请求
- 资源管理

完整部署请使用支持 PHP 的服务器环境。

## 常见问题

### 1. 为什么后台保存失败？

请检查 `data/` 目录是否有写入权限。

可以尝试：

```bash
chmod -R 775 data
```

---

### 2. 为什么图标上传失败？

请检查上传目录是否存在：

```text
uploads/icons/
```

并确保该目录有写入权限。

---

### 3. 为什么后台登录不了？

请检查：

```text
config.php
```

确认后台账号和密码是否正确。

---

### 4. 可以部署到宝塔吗？

推荐宝塔部署。

宝塔部署方式：

```text
1. 新建 PHP 网站
2. 上传项目文件
3. 设置运行目录为项目根目录
4. 确认 PHP 版本
5. 设置 data 和 uploads 目录权限
6. 访问 /admin.php 登录后台
```

---

### 5. 需要数据库吗？

不需要。

本项目使用 JSON 文件存储数据，适合轻量级资源导航站。
